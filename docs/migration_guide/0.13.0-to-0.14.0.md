# Upgrading from nlink 0.13.0 to 0.14.0

0.14.0 is a **mostly additive** release. The bulk of the work is
new typed APIs that supersede older patterns without forcing a
migration today. There's exactly one deprecation to plan around,
and that's the `tc::builders::*` family — which is **deleted in
0.15.0**, so it's worth migrating in the 0.14.0 cycle if you can.

## Breaking removals

None. 0.14.0 keeps every public API from 0.13.0 working.

## Behaviour changes

None at the API level. New deprecation warnings (see below)
will appear at compile time if you reach for the deprecated
modules.

## New deprecations

### `nlink::tc::builders::*` and `nlink::tc::options::*`

The legacy string-args TC builders and per-kind options modules
gained `#[deprecated]` markers in 0.14.0. Code that used them
keeps working but emits a compile-time warning. **The actual
deletion ships in 0.15.0** — see
[`0.14.0-to-0.15.0.md`](0.14.0-to-0.15.0.md) for the migration
table.

If you scope `#[allow(deprecated)]` around the calls, you defer
the work but you *will* hit it at the 0.15.0 upgrade.

```rust
// Compile-warns in 0.14.0
use nlink::tc::builders::qdisc as qdisc_builder;
qdisc_builder::add(conn, dev, parent, handle, kind, &params).await?;

// Migrate now to skip the 0.15.0 churn
use nlink::netlink::tc::HtbQdiscConfig;
use nlink::{ParseParams, TcHandle};
let cfg = HtbQdiscConfig::parse_params(&params)?;
conn.add_qdisc_full(dev, parent, handle, cfg).await?;
```

The same shape works for `class`, `filter`, and the per-kind
`options::*` modules. See the 0.15.0 guide's full migration
table.

## Notable additive changes

### Typed-units `parse_params` rollout (15 slices, 25 parsers)

Every typed TC qdisc + filter config gained a
`parse_params(&[&str]) -> Result<Self>` method:

```rust
use nlink::netlink::tc::NetemConfig;

let cfg = NetemConfig::parse_params(&[
    "delay", "100ms", "10ms", "25%",
    "loss", "1%",
])?;
```

Strict parsing — unknown tokens / missing values / unparseable
inner values return `Error::InvalidMessage("netem: ...")`
instead of being silently swallowed (which is what the legacy
parsers did). 18 qdisc + 7 filter parsers shipped in 0.14.0;
the remaining 2 filter kinds (`u32`, `basic`) and all 14 action
kinds ship in 0.15.0.

`bins/tc qdisc add` migrated to typed dispatch for every kind it
supports; filter side at 7 of 9 in 0.14.0 (full coverage in
0.15.0).

### Reconcile pattern

Typed reconcile helpers for declarative network state:

```rust
use nlink::{ReconcileOptions, ReconcileReport};

let report: ReconcileReport = network_config
    .reconcile(&conn, ReconcileOptions::default())
    .await?;
println!("{} added, {} removed, {} unchanged",
    report.added.len(), report.removed.len(), report.unchanged);
```

See the `nlink::netlink::config` module for the full surface.

### Plan 137 PR A — typed ctnetlink mutation on `Connection<Netfilter>`

```rust
use nlink::netlink::netfilter::{ConntrackBuilder, ConntrackStatus};

let entry = ConntrackBuilder::new()
    .family(libc::AF_INET as u8)
    .orig_src("10.0.0.1".parse()?)
    .orig_dst("10.0.0.2".parse()?)
    .orig_proto(6)            // TCP
    .orig_src_port(12345)
    .orig_dst_port(80)
    .timeout(120)
    .status(ConntrackStatus::CONFIRMED);
conn.add_conntrack(entry).await?;
```

Plus `update_conntrack`, `del_conntrack`, `flush_conntrack`,
`get_conntrack`. See `docs/recipes/conntrack-programmatic.md`.

### Plan 137 PR B — multicast event subscription

`Connection<Netfilter>` now exposes `subscribe_conntrack_events()`
returning a typed `Stream<ConntrackEvent>`. See the same recipe
for the events flow.

### Six new recipes in `docs/recipes/`

Plan 135 PR B shipped 6 of 7 planned recipes:

- `per-peer-impairment` — per-destination netem on shared L2
- `bridge-vlan` — VLAN-aware bridge, trunk/access ports
- `bidirectional-rate-limit` — HTB egress + IFB ingress
- `wireguard-mesh` — 3-node mesh in `nlink::lab` namespaces
- `multi-namespace-events` — `StreamMap` fan-in across N namespaces
- `conntrack-programmatic` — ctnetlink mutation + multicast events
- `nftables-stateful-fw` — table/chain/rule plumbing + atomic
  transactions

The two deferred recipes (`xfrm-ipsec-tunnel`,
`cgroup-classification`) ship in 0.15.0 / post-0.15.0.

### `MacsecLink` rtnetlink builder

Typed builder for MACsec virtual interfaces:

```rust
use nlink::netlink::link::MacsecLink;
let link = MacsecLink::new("macsec0")
    .link_underlying("eth0")
    .sci(0x0102_0304_0506_0708)
    .encrypt(true)
    .build();
conn.add_link(link).await?;
```

`examples/genl/macsec.rs` was migrated to use it directly.
