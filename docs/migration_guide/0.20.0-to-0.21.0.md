# Upgrading from 0.20.x to 0.21.0

0.20.0 shipped as an emergency wire-format fix; 0.20.1 added the
typed-API tightening (`AddressFamily`, `ChainName`, typed
`Percent` on declarative builders, `RuleMessage` accessors, DPLL
`sint` runtime) **additively** with one-release deprecations.
0.21.0 closes the cycle: deprecated APIs are removed, the
typed-API surface is the only surface, and a handful of audit
plans deferred from 0.20.1 land.

This is a breaking release. `cargo-semver-checks` against 0.20.1
intentionally lists each removal below.

If you're upgrading directly from 0.19.x, also read
[`0.18.0-to-0.19.0.md`](0.18.0-to-0.19.0.md) and the
[`[0.20.0]`](../../CHANGELOG.md) CHANGELOG section.

## Headline removals

### 1. `DpllPin::fractional_frequency_offset_ppt` widened to `Option<i64>`

The kernel emits this DPLL pin attribute as a variable-length
`sint` (4 bytes if the value fits in `s32`, 8 bytes otherwise).
0.20.1 routed the parse through `parse_sint_attr_as_i32`, which
*correctly errored* on overflowing 8-byte payloads but **failed
the entire `DpllPinReply::from_bytes` parse** rather than
truncating. SyncE bring-up traffic routinely emits 8-byte FFOs
that overflow `i32`; pre-0.21 callers either lived with the
parse-error or used the escape-hatch
`DpllPinReply::ffo_ppt_i64_from_payload(&[u8])`.

0.21 widens the field type so the full-width value parses
cleanly into the struct.

```rust
// Before (0.20.x):
let ffo: Option<i32> = pin.fractional_frequency_offset_ppt;

// After (0.21):
let ffo: Option<i64> = pin.fractional_frequency_offset_ppt;
```

Migration: anywhere you cast the field via `as i32`, drop the
cast (you already have an `i64`). Anywhere you stored the value
in an `i32` variable, widen. The escape-hatch helper stays
available; it's now informational since the struct field carries
the same value.

### 2. Route-rule API: raw-`u8` family removed

The three methods deprecated in 0.20.1:

- `Connection::<Route>::flush_rules(family: u8)`
- `Connection::<Route>::get_rules_for_family(family: u8)`
- `Connection::<Route>::del_rule_by_priority(family: u8, _)`

are gone. The typed siblings (`*_typed`, taking `AddressFamily`)
are renamed back to the original names — there's only one form
now and it's the safe one.

```rust
use nlink::AddressFamily;

// Before (0.20.x):
conn.flush_rules(libc::AF_INET as u8).await?;
conn.get_rules_for_family(libc::AF_INET6 as u8).await?;
conn.del_rule_by_priority(libc::AF_INET as u8, 100).await?;

// After (0.21):
conn.flush_rules(AddressFamily::v4()).await?;
conn.get_rules_for_family(AddressFamily::v6()).await?;
conn.del_rule_by_priority(AddressFamily::v4(), 100).await?;
```

`AddressFamily::from_raw(u8)` exists for `&str` parsing callers
who need to round-trip raw bytes through the type.

### 3. `QdiscBuilder::loss(f64)` removed

The declarative netem `.loss(f64)` setter is gone. Use
`.loss_pct(Percent)` with the typed argument:

```rust
use nlink::util::Percent;

// Before (0.20.x):
QdiscBuilder::new("eth0").netem().loss(1.5);

// After (0.21) — percent value:
QdiscBuilder::new("eth0").netem().loss_pct(Percent::new(1.5));

// After (0.21) — fraction:
QdiscBuilder::new("eth0").netem().loss_pct(Percent::from_fraction(0.015));
```

The raw-`f64` form silently accepted out-of-range and NaN values.
`Percent::new` clamps to `[0, 100]` and forces the
percent-vs-fraction distinction at the call site.

**Plan 228 extension — parity setters.** The declarative
`QdiscBuilder` gains five additional netem setters mirroring the
imperative `NetemConfig`:

- `duplicate_pct(Percent)`
- `corrupt_pct(Percent)`
- `reorder_pct(Percent)`
- `loss_correlation_pct(Percent)`
- `delay_correlation_pct(Percent)`

The `DeclaredQdiscType::Netem` enum variant gains corresponding
`Option<f64>` fields. Pattern-matching consumers need to either
list the new fields or use the `..` rest pattern.

### 4. `Verdict::Jump(String)` / `Verdict::Goto(String)` removed

The two `String`-shaped `Verdict` variants are gone. Use the
typed `ChainName`-carrying variants:

```rust
use nlink::netlink::nftables::types::{Verdict, ChainName};

// Before (0.20.x):
let v = Verdict::Jump("input_filter".to_string());

// After (0.21):
let v = Verdict::JumpTo(ChainName::new("input_filter")?);
```

`ChainName::new` validates the kernel chain-name contract
(non-empty, no interior NUL, `len <= NFT_NAME_MAXLEN-1`) at
construction. Pre-0.21 a bad name went all the way to the kernel
and surfaced as a generic apply-time error.

**`RuleBuilder::jump` / `goto` reworked.** The 0.20.1 infallible
shim that fell back to the deprecated `String` variants on bad
names is gone. The 0.21 form takes a pre-validated `ChainName`
and stays infallible:

```rust
// Before (0.20.x), infallible-on-bad-name (kernel rejects later):
rule.jump("name");
rule.goto("other");

// After (0.21), if you already have a ChainName:
rule.jump(ChainName::new("name")?);
rule.goto(ChainName::new("other")?);

// After (0.21), &str form returning Result:
rule.try_jump("name")?;
rule.try_goto("other")?;
```

### 5. `RuleMessage` field-visibility flip + `#[non_exhaustive]`

Plan 231 closeout. Six `*Message` types in
`crates/nlink/src/netlink/messages/` gain `#[non_exhaustive]`:
`AddressMessage`, `LinkMessage`, `NeighborMessage`,
`NsIdMessage`, `RouteMessage`, `TcMessage`. `RuleMessage` and
`NsIdMessage` additionally have their `pub` fields flipped to
`pub(crate)`. The 0.20.1 accessor batch shipped per-field
methods additively (with both pub fields *and* accessors
visible); 0.21 closes the convention by hiding the fields.

```rust
// Before (0.20.x), direct field access:
let pri = rule.priority;
let tbl = rule.table;
let src = rule.source;

// After (0.21), accessor methods:
let pri = rule.priority();
let tbl = rule.table_id();
let src = rule.source();
```

The accessor methods existed in 0.20.1; only the field-access
form is removed. New audit gate
`scripts/audit-message-accessor-convention.sh` keeps the
convention closed against drift.

### 6. Sibling parsed-result types — Plan 231 extension

The four parsed-result types the 0.20.1 batch flagged but only
RuleMessage got the full treatment in 0.20.1: `BridgeVlanEntry`,
`FdbEntry`, `MplsRoute`, `Nexthop`, plus the
`NexthopGroupMember` sub-type. All five flip from `pub` fields
to `pub(crate)` + `#[non_exhaustive]` and gain per-field
accessors.

```rust
// Before (0.20.x):
let ifindex = entry.ifindex;
let vlan = entry.vlan;
let dst = entry.dst.as_ref();

// After (0.21):
let ifindex = entry.ifindex();
let vlan = entry.vlan();
let dst = entry.dst();          // Option<IpAddr> (Copy now returns by value)
```

Builders that need to construct a sibling type for testing or
reconstruction use the new constructor:

```rust
use nlink::netlink::nexthop::NexthopGroupMember;

// Before (0.20.x):
let m = NexthopGroupMember { id: 7, weight: 1 };

// After (0.21):
let m = NexthopGroupMember::new(7, 1);
```

## CI gates added

- **`audit-message-accessor-convention`** — fails the build if
  any `*Message` struct in `crates/nlink/src/netlink/messages/`
  has bare `pub` fields or is missing `#[non_exhaustive]`. Also
  covers `BridgeVlanEntry` / `FdbEntry` / `MplsRoute` /
  `Nexthop` in their respective files. Self-test under
  `scripts/test-audit-message-accessor-convention.sh` (Plan 237
  pattern).
- **`audit-recipe-drift`** — fails the build if recipe markdown
  under `docs/recipes/` contains known-stale API patterns
  (sync `conn.events()`, `Verdict::Jump("…")`, `Hook::Ingress`,
  `with_purge`, …).
- **`doctest-nlink`** — runs `cargo test --doc -p nlink`.
  Non-blocking for one cycle via `continue-on-error: true` per
  Plan 229 §3 rollout; promote to blocking once one cycle's PRs
  merge with it green.

## Quick-fix cheat sheet

| Pattern | Replacement |
|---|---|
| `pin.fractional_frequency_offset_ppt as i32` | drop the `as i32` (field is `i64` now) |
| `conn.flush_rules(libc::AF_INET as u8)` | `conn.flush_rules(AddressFamily::v4())` |
| `conn.get_rules_for_family(fam as u8)` | `conn.get_rules_for_family(AddressFamily::from_raw(fam))` |
| `conn.del_rule_by_priority(fam, p)` | `conn.del_rule_by_priority(AddressFamily::from_raw(fam), p)` |
| `.loss(1.5)` | `.loss_pct(Percent::new(1.5))` |
| `Verdict::Jump("name")` | `Verdict::JumpTo(ChainName::new("name")?)` |
| `Verdict::Goto("name")` | `Verdict::GotoTo(ChainName::new("name")?)` |
| `.jump("name")` (RuleBuilder) | `.try_jump("name")?` or `.jump(ChainName::new("name")?)` |
| `.goto("name")` (RuleBuilder) | `.try_goto("name")?` or `.goto(ChainName::new("name")?)` |
| `rule.priority` / `rule.table` | `rule.priority()` / `rule.table_id()` |
| `rule.source` / `rule.destination` | `rule.source()` / `rule.destination()` |
| `entry.ifindex` (FdbEntry / VLAN / …) | `entry.ifindex()` |
| `nh.gateway` / `nh.blackhole` | `nh.gateway()` / `nh.is_blackhole()` |
| `route.label.0` (MplsRoute) | `route.label().0` |
| `NexthopGroupMember { id, weight }` | `NexthopGroupMember::new(id, weight)` |
| `DeclaredQdiscType::Netem { .., loss_percent, limit }` (pattern) | add the 5 new fields or use `..` rest |
