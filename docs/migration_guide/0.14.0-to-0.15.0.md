# Upgrading from nlink 0.14.0 to 0.15.0

0.15.0 is a **major release**. The `#[deprecated]` markers
introduced in 0.14.0 have been redeemed: the modules they pointed
at are gone. If you took the deprecation warnings seriously and
migrated in the 0.14.0 cycle, the upgrade is mechanical. If you
ignored them, you have work to do.

The single concrete change to plan around: **`nlink::tc::builders`
and `nlink::tc::options` are deleted entirely**. Every typed
replacement is already in the 0.14.0 codebase, so a migration
that compiles against 0.14.0 will compile against 0.15.0.

## Breaking removals

### `nlink::tc::builders::class` → `Connection::add_class_config`

```rust
// 0.14.0 (deprecated)
use nlink::tc::builders::class as class_builder;
class_builder::add(conn, "eth0", "1:", "1:10", "htb",
    &["rate", "100mbit", "ceil", "1gbit"]).await?;

// 0.15.0
use nlink::netlink::tc::HtbClassConfig;
use nlink::{ParseParams, TcHandle};

let cfg = HtbClassConfig::parse_params(
    &["rate", "100mbit", "ceil", "1gbit"]
)?;
conn.add_class_config(
    "eth0",
    TcHandle::major_only(1),
    TcHandle::new(1, 0x10),
    cfg,
).await?;
```

Same shape for `change_class_config` / `replace_class_config` /
`del_class`.

### `nlink::tc::builders::qdisc` → `Connection::add_qdisc_full`

```rust
// 0.14.0
use nlink::tc::builders::qdisc as qdisc_builder;
qdisc_builder::add(conn, "eth0", "root", Some("1:"), "htb",
    &["default", "10"]).await?;

// 0.15.0
use nlink::netlink::tc::HtbQdiscConfig;
use nlink::{ParseParams, TcHandle};

let cfg = HtbQdiscConfig::parse_params(&["default", "10"])?;
conn.add_qdisc_full(
    "eth0",
    TcHandle::ROOT,
    Some(TcHandle::major_only(1)),
    cfg,
).await?;
```

18 typed qdisc configs cover every kind the legacy builder
supported: `HtbQdiscConfig`, `NetemConfig`, `CakeConfig`,
`TbfConfig`, `SfqConfig`, `PrioConfig`, `FqCodelConfig`,
`RedConfig`, `PieConfig`, `HfscConfig`, `DrrConfig`, `QfqConfig`,
`IngressConfig`, `ClsactConfig`, `PlugConfig`, `MqprioConfig`,
`EtfConfig`, `TaprioConfig`.

### `nlink::tc::builders::filter` → `Connection::add_filter_full`

```rust
// 0.14.0
use nlink::tc::builders::filter as filter_builder;
filter_builder::add(conn, "eth0", "1:", "ip", Some(100),
    "flower", &["dst_port", "80", "ip_proto", "tcp"]).await?;

// 0.15.0
use nlink::netlink::filter::FlowerFilter;
use nlink::{FilterPriority, ParseParams, TcHandle};

let cfg = FlowerFilter::parse_params(
    &["dst_port", "80", "ip_proto", "tcp"]
)?;
conn.add_filter_full(
    "eth0",
    TcHandle::major_only(1),
    None,           // handle (kernel assigns)
    0x0800,         // ETH_P_IP
    FilterPriority::new(100).into_raw(),
    cfg,
).await?;
```

9 typed filter configs: `FlowerFilter`, `U32Filter`,
`MatchallFilter`, `FwFilter`, `RouteFilter`, `BpfFilter`,
`CgroupFilter`, `FlowFilter`, `BasicFilter`.

### `nlink::tc::builders::filter::parse_protocol` → inline

The 10-line lookup table is now private to `bins/tc`. If you
relied on it, copy it (it's the kind of helper that doesn't merit
a public API surface):

```rust
fn parse_protocol_u16(s: &str) -> Result<u16, nlink::Error> {
    Ok(match s.to_lowercase().as_str() {
        "all" => 0x0003,
        "ip" => 0x0800,
        "ipv6" => 0x86DD,
        "arp" => 0x0806,
        "802.1q" | "vlan" => 0x8100,
        "802.1ad" => 0x88A8,
        "mpls_uc" => 0x8847,
        "mpls_mc" => 0x8848,
        _ => {
            if let Some(hex) = s.strip_prefix("0x") {
                u16::from_str_radix(hex, 16).map_err(|_| {
                    nlink::Error::InvalidMessage(
                        format!("invalid protocol: {s}")
                    )
                })?
            } else {
                s.parse().map_err(|_| {
                    nlink::Error::InvalidMessage(
                        format!("unknown protocol: {s}")
                    )
                })?
            }
        }
    })
}
```

(If a downstream user files an issue asking for this on the
public surface, it can ship in a future minor release.)

### `nlink::tc::builders::action` → `Connection::add_action`

This one's new in 0.15.0 — Plan 139 PR A added typed CRUD for
standalone shared actions:

```rust
// 0.14.0
use nlink::tc::builders::action as action_builder;
action_builder::add(conn, "gact", &["drop"]).await?;

// 0.15.0
use nlink::netlink::action::GactAction;
use nlink::ParseParams;

let action = GactAction::parse_params(&["drop"])?;
conn.add_action(action).await?;
// or build directly:
conn.add_action(GactAction::drop()).await?;
```

Plus `del_action(kind, index)`, `get_action(kind, index)`,
`dump_actions(kind)`. 14 typed action kinds: `GactAction`,
`MirredAction`, `PoliceAction`, `VlanAction`, `SkbeditAction`,
`ConnmarkAction`, `CsumAction`, `SampleAction`,
`TunnelKeyAction`, `NatAction`, `SimpleAction`, `BpfAction`,
`CtAction`, `PeditAction`.

> **`PeditAction::parse_params` is a stub** that always rejects
> with "use the typed builder" — `tc(8)`'s pedit DSL is genuinely
> complex and is "punt-eligible until a downstream user asks"
> per Plan 139 §10. The typed setters (`set_ipv4_src`,
> `set_tcp_dport`, etc.) work normally.

### `nlink::tc::handle` → `nlink::TcHandle`

```rust
// 0.14.0
use nlink::tc::handle::parse_handle;
let raw_u32: u32 = parse_handle("1:a").unwrap();

// 0.15.0
use nlink::TcHandle;
let h: TcHandle = "1:a".parse()?;       // typed
let raw_u32: u32 = h.as_raw();          // if you really need the u32
```

`TcHandle` is the canonical typed surface; `Display` /
`FromStr` round-trip the `tc(8)` `<major>:<minor>` notation, and
inspection helpers (`is_root()`, `is_ingress()`, etc.) exist on
the type.

### `nlink::tc::options::*` → typed `*Config::parse_params`

Each kind-specific options module is replaced by the matching
typed config's `parse_params`:

| Removed | Replacement |
|---|---|
| `tc::options::cake::build` | `CakeConfig::parse_params(&[..])?` |
| `tc::options::codel::build` | `CodelConfig` is shipped as part of `FqCodelConfig`; pass `[..]` to `FqCodelConfig::parse_params` |
| `tc::options::fq_codel::build` | `FqCodelConfig::parse_params(&[..])?` |
| `tc::options::fq::build` | (no typed equivalent today — file an issue if needed) |
| `tc::options::htb::build` | `HtbQdiscConfig::parse_params(&[..])?` |
| `tc::options::netem::build` | `NetemConfig::parse_params(&[..])?` |
| `tc::options::prio::build` | `PrioConfig::parse_params(&[..])?` |
| `tc::options::sfq::build` | `SfqConfig::parse_params(&[..])?` |
| `tc::options::tbf::build` | `TbfConfig::parse_params(&[..])?` |

### The `tc` Cargo feature

```toml
# 0.14.0
nlink = { version = "0.14", features = ["tc", "output"] }

# 0.15.0
nlink = { version = "0.15", features = ["output"] }
```

The `tc` feature only gated the deleted modules. `nlink::TcHandle`
and the typed configs live in always-built modules; nothing else
to enable.

## Behaviour changes

### `parse_params` strictness

Every typed `parse_params` rejects unknown tokens / missing values
/ unparseable inner values with `Error::InvalidMessage("kind:
...")`. The legacy parsers silently skipped unknown tokens.

If your code passed token slices that the legacy parser quietly
ignored (typos in token names, extra positional args, etc.), the
typed parser will now error. Fix the input — that's the point of
the typed surface.

### `bins/tc` unknown-kind behaviour

```text
# 0.14.0
$ tc qdisc add dev eth0 root unknown_kind
# (no error, kernel rejects with EINVAL — confusing)

# 0.15.0
$ tc qdisc add dev eth0 root unknown_kind
Error: tc qdisc: unknown kind `unknown_kind` (recognised: htb, netem, cake, tbf, sfq, prio, fq_codel, red, pie, hfsc, drr, qfq, ingress, clsact, plug, mqprio, etf, taprio)
```

Same for `tc filter add` and `tc action add`.

### `tc filter del` requires the full lookup tuple

```text
# 0.14.0 — partial spec ok
$ tc filter del dev eth0          # delete all filters on eth0

# 0.15.0 — full tuple required
$ tc filter del dev eth0
Error: tc filter del: --protocol is required (typed del needs the full lookup tuple)

$ tc filter del dev eth0 --protocol ip --prio 100  # works
```

The typed `Connection::del_filter` takes `(parent, protocol, prio)`
as a unit. Users who want partial-spec deletion should
`tc filter show dev eth0` to enumerate, then `del` by tuple. (Or
file an issue if a typed flush-by-partial-spec helper would be
useful.)

### `tc action del` requires `--index`

```text
# 0.14.0
$ tc action del gact          # interactive prompt or kernel-default

# 0.15.0
$ tc action del gact          # error: --index is required
$ tc action del gact --index 7
```

## New deprecations

None — 0.15.0 is the redemption release for the 0.14.0
deprecations. The post-0.15.0 cycle starts with a clean
deprecation slate.

## Notable additive changes

### Typed XFRM SA + SP CRUD on `Connection<Xfrm>`

`Connection<Xfrm>` was dump-only in 0.14.0. 0.15.0 adds the full
write path:

```rust
use nlink::netlink::{Connection, Xfrm};
use nlink::netlink::xfrm::{
    XfrmSaBuilder, XfrmSpBuilder, XfrmUserTmpl, XfrmSelector,
    IpsecProtocol, XfrmMode, PolicyDirection,
};

let conn = Connection::<Xfrm>::new()?;

let sa = XfrmSaBuilder::new(
    "10.0.0.1".parse()?, "10.0.0.2".parse()?,
    0xdead_beef, IpsecProtocol::Esp,
)
.mode(XfrmMode::Tunnel)
.reqid(42)
.aead_aes_gcm(&[0u8; 20], 128);
conn.add_sa(sa).await?;

let sp = XfrmSpBuilder::new(
    XfrmSelector { family: libc::AF_INET as u16, ..Default::default() },
    PolicyDirection::Out,
)
.priority(100)
.template(XfrmUserTmpl::match_any(
    "10.0.0.1".parse()?, "10.0.0.2".parse()?,
    IpsecProtocol::Esp, XfrmMode::Tunnel, 42,
));
conn.add_sp(sp).await?;
```

Plus `update_sa`/`del_sa`/`flush_sa`/`flush_sa_proto`/`get_sa`
and the same set on the `_sp` side. 20 wire-format unit tests
round-trip through the existing dump-side parsers.

### Sealed `nlink::ParseParams` trait

41 typed configs (18 qdisc + 9 filter + 14 action) implement the
sealed `ParseParams` trait. Generic dispatch is now possible:

```rust
use nlink::ParseParams;

fn parse<C: ParseParams>(params: &[&str]) -> nlink::Result<C> {
    C::parse_params(params)
}
let cfg: HtbQdiscConfig = parse(&["default", "10"])?;
```

Sealed via `private::Sealed`; third-party crates can use the
trait but cannot implement it. The contract (strict rejection,
kind-prefixed error messages) is intentionally narrow.

### `nlink::lab::has_module` + `require_module!` macros

For integration tests that depend on optional kernel features:

```rust
#[tokio::test]
async fn needs_conntrack() -> nlink::Result<()> {
    nlink::require_root!();
    nlink::require_module!("nf_conntrack");
    nlink::require_module!("nf_conntrack_netlink");
    // ... real test ...
    Ok(())
}
```

`has_module` checks `/sys/module/<name>` so it works for both
loadable modules (loaded via `modprobe`) and built-in features.

## Worked example — full HTB tree migration

```rust
// 0.14.0
use nlink::tc::builders::{class, qdisc};

#[allow(deprecated)]
async fn setup_htb_0_14(conn: &Connection<Route>) -> Result<()> {
    qdisc::add(conn, "eth0", "root", Some("1:"), "htb",
        &["default", "10"]).await?;
    class::add(conn, "eth0", "1:", "1:1", "htb",
        &["rate", "1gbit", "ceil", "1gbit"]).await?;
    class::add(conn, "eth0", "1:1", "1:10", "htb",
        &["rate", "100mbit", "ceil", "500mbit"]).await?;
    Ok(())
}
```

```rust
// 0.15.0
use nlink::netlink::tc::{HtbClassConfig, HtbQdiscConfig};
use nlink::{ParseParams, TcHandle};

async fn setup_htb_0_15(conn: &Connection<Route>) -> Result<()> {
    conn.add_qdisc_full(
        "eth0",
        TcHandle::ROOT,
        Some(TcHandle::major_only(1)),
        HtbQdiscConfig::parse_params(&["default", "10"])?,
    ).await?;
    conn.add_class_config(
        "eth0",
        TcHandle::major_only(1),
        TcHandle::new(1, 1),
        HtbClassConfig::parse_params(
            &["rate", "1gbit", "ceil", "1gbit"]
        )?,
    ).await?;
    conn.add_class_config(
        "eth0",
        TcHandle::new(1, 1),
        TcHandle::new(1, 0x10),
        HtbClassConfig::parse_params(
            &["rate", "100mbit", "ceil", "500mbit"]
        )?,
    ).await?;
    Ok(())
}
```

Or skip the parse-params indirection entirely if your call sites
don't need string-driven config:

```rust
use nlink::{Bytes, Rate, TcHandle};

async fn setup_htb_typed(conn: &Connection<Route>) -> Result<()> {
    conn.add_qdisc_full(
        "eth0",
        TcHandle::ROOT,
        Some(TcHandle::major_only(1)),
        HtbQdiscConfig::new().default_class(0x10).build(),
    ).await?;
    conn.add_class_config(
        "eth0",
        TcHandle::major_only(1),
        TcHandle::new(1, 1),
        HtbClassConfig::new(Rate::gbit(1))
            .ceil(Rate::gbit(1))
            .build(),
    ).await?;
    conn.add_class_config(
        "eth0",
        TcHandle::new(1, 1),
        TcHandle::new(1, 0x10),
        HtbClassConfig::new(Rate::mbit(100))
            .ceil(Rate::mbit(500))
            .build(),
    ).await?;
    Ok(())
}
```

The typed `Rate` constructor accepts `kbit`/`mbit`/`gbit` /
`bytes_per_sec` / `bits_per_sec` and round-trips with `tc(8)`
strings via `Display` / `FromStr`. Same for `Bytes` (sizes) and
`Percent` (probabilities).
