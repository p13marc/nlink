## Plan 187 — `Error` API sign normalization + Io-shape predicate fix

### What changed

- **Factory sign normalization**: `Error::from_errno`,
  `from_errno_with_context`, `from_errno_ext_ack`,
  `from_errno_with_context_ext_ack` now call `.abs()` on the
  input. Both `from_errno(1)` and `from_errno(-1)` produce
  stored `errno = 1`.

- **Predicate Io-shape fix**: `Error::errno()` unwraps
  `Error::Io(io_err)` via `raw_os_error()` so every predicate
  goes through one code path. Three predicates that previously
  matched on the variant directly (`is_busy`,
  `is_already_exists`, `is_permission_denied`) now go through
  `errno()` and catch the Io shape too. Critical for
  `apply_reconcile` retry classification.

### Migration steps

**For most consumers:** no action required. Production code
using `?`-bubbled kernel errors keeps working.

**For direct tests / mocks** that asserted `Some(-1)` after
calling `from_errno_ext_ack(1, ..)`:

```rust
// Before 0.19
assert_eq!(err.errno(), Some(-1));

// 0.19+
assert_eq!(err.errno(), Some(1));
```

**For boxed-source wrappers**: still a footgun until Plan 187
phase 2 (chain_walk helper) lands. Until then, work around by
unwrapping the box manually:

```rust
let inner = source.downcast_ref::<Box<nlink::Error>>()
    .map(|b| b.as_ref())
    .or_else(|| source.downcast_ref::<nlink::Error>());
```

**Phase 2 shipped**: `Error::chain_walk` + `root_cause` +
`contexts` now handle both shapes:

```rust
use nlink::Error;
// Find the first ENOBUFS anywhere in the chain.
let enobufs = Error::chain_walk(&outer_err).find(|e| e.is_no_buffer_space());
// Get the deepest nlink error.
let bottom = outer_err.root_cause();    // when outer_err is &nlink::Error
// All nlink layers as a Vec.
let layers: Vec<_> = err.contexts();
```

If you've been writing the manual ladder, switch to `chain_walk`.

## Plan 188 — `ConfigDiff::apply` + `RouteBuilder::default_v{4,6}` + `summary()` deprecation (Phase 1)

### What changed

- New `ConfigDiff::apply(&conn, opts)` inherent method
  mirroring `NftablesDiff::apply`.
- New `RouteBuilder::default_v4()` / `default_v6()`
  constructors mirroring `Ipv4Route::default_route()` /
  `Ipv6Route::default_route()` (Plan 184).
- `ConfigDiff::summary()` and `NftablesDiff::summary()`
  deprecated in favor of the `Display` impl. Removed in 0.20.

### Migration steps

```rust
// Before (still works in 0.19; warned)
let s = diff.summary();
println!("{}", s);

// After
let s = diff.to_string();
// OR
println!("{diff}");
```

### All Plan 188 phases complete (7 of 7 sub-items)

`apply_reconcile` shipped with the others; usage:

```rust
use nlink::netlink::nftables::config::ReconcileOptions;
let report = cfg.apply_reconcile(
    &conn,
    ReconcileOptions::default().max_retries(5),
).await?;
if report.attempts > 1 {
    tracing::warn!(retries = report.attempts - 1, "transient conflict");
}
```

Note the import path — Plan 188 §2.4 reuses the nftables-side
`ReconcileOptions` rather than introducing a third with the
same name.

## Plan 193 — Parser robustness (Phase 1 — policy + CI gate)

### What changed

The defensive-parsing conventions used across the lib are now
documented in CLAUDE.md under §"Parser robustness" (three
rules: accept-larger-than-expected, pathological-length input
guards, recoverable per-message parse failures) and pinned by
a new CI gate (`scripts/audit-recv-loop-error-handling.sh`).

### Migration steps

**No consumer action required.** The lib already follows the
three rules; the policy doc + CI gate prevent future drift.

Contributors writing new event parsers (`impl EventSource
for *`) must use `MessageIter::new(data).flatten()` or the
`let Ok(...) = msg_result else { continue }` pattern — not
the `?` operator. The audit script catches violations.

### What's still ahead in Plan 193 (later commits)

- §2.1 size-check sweep — N/A, the lib already uses the
  defensive `< SIZE` form everywhere
- §2.2 multipath/nexthop pathological-input guards — N/A, no
  current parser to guard (multipath is write-only in nlink
  today; documented as a known parser gap)
- §2.4 fuzz target setup — deferred to a later commit;
  requires nightly Rust + the fuzz/ subdirectory
