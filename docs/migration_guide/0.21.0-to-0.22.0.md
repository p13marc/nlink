# Upgrading from 0.21.0 to 0.22.0

**Purely additive for library consumers.** `cargo-semver-checks`
against 0.21.0 reports no breaking changes — every new typed config,
`Connection` method, and module landed additively. If you depend on
the `nlink` crate, bump the version and nothing else changes.

The one change anyone running the demo **binaries** will notice is a
rename (see [Behaviour change](#behaviour-change--demo-binaries-renamed-to-nlink-)
below).

If you're upgrading from an older release, read the intervening guides
too — start at [`README.md`](README.md).

## Behaviour change — demo binaries renamed to `nlink-*`

The eight demo binaries that previously built under the bare
system-tool name now carry the `nlink-` prefix, matching the
already-prefixed `nlink-config` / `nlink-ethtool` / `nlink-diag`:

| Before | After |
|---|---|
| `ip` | `nlink-ip` |
| `tc` | `nlink-tc` |
| `ss` | `nlink-ss` |
| `nft` | `nlink-nft` |
| `bridge` | `nlink-bridge` |
| `wg` | `nlink-wg` |
| `wifi` | `nlink-wifi` |
| `devlink` | `nlink-devlink` |

Only the produced binary name changed — the Cargo **package** names
were already `nlink-*`, so `cargo run -p nlink-ip` is unaffected. If
you invoked a built artifact directly, switch the path:

```bash
# Before:
./target/release/ip link show

# After:
./target/release/nlink-ip link show
```

The rename keeps a demo binary from shadowing the real
iproute2/iw/ethtool tool on `PATH`. This is the only migration action
in 0.22.0.

## What's new (additive)

0.22.0 is a large feature release — the proof-of-concept binaries were
hardened end-to-end and the library coverage gaps they surfaced were
closed. See `CHANGELOG.md ## [0.22.0]` for the full list; highlights:

### Demo-binary hardening (epic #30)

Every demo binary had its silent no-ops, dead flags, and unexposed
library APIs fixed: `ip` (macsec add/set/del, `netns set <nsid>`,
namespace-safe `neighbor`/`maddr` show, strict link-add parsing),
`tc` (`qdisc show --invisible`, filter `--chain` + show-across-parents,
namespace-safe show), `ss` (`-O/--oneline`, kernel-side port filters,
non-inet expr fix), `bridge` (`monitor`, `fdb --extern-learn`/`--brport`),
`wg` (`addconf`/`setconf`/`syncconf` config-file apply, `--json`),
`devlink` (type-aware `param set`), `nft` (idempotent deletes, richer
`list rules`), `wifi` (richer station detail).

### Library capability gaps (#29)

New typed surfaces backing the bin work: tc qdiscs `cbs`, `multiq`,
`hhf`, `sfb`, `gred`, `dsmark`, `skbprio`; tc filter `tcindex`; tc
actions `mpls`, `skbmod`; ethtool WoL / EEE / FEC get-set; nl80211
`del_station`; AF_PACKET socket diagnostics.

### TC / ethtool / sockdiag coverage epic (#115)

The deferred-backlog kinds, each with a strict `parse_params` and bin
dispatch: tc qdiscs `choke`, `pfifo_fast`, `gred`, `atm` (#116); tc
filter `rsvp` / `rsvp6` (#117); tc actions `ctinfo`, `ife`, `gate`
(#118); ethtool FEC setter + module-EEPROM read + RSS read (#119);
sockdiag `INET_DIAG_REQ_BYTECODE` kernel-side port filtering (#120).

### Declarative config (#108, #109)

`NetworkConfig` gains a validating `serde` round-trip (CIDR/MAC strings
parse-validated via `try_from`); the `nft` bin gains declarative
`reconcile` / `diff` over `NftablesConfig`.
