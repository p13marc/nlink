# Plan 145 — 0.15.1 doc warning cleanup

## 1. Summary

`cargo doc -p nlink --no-deps` emits **7 warnings** for
unresolved intra-doc cross-references in 0.15.0. All are visible
on docs.rs right now, where they render as plain code-formatted
text instead of clickable links. Three distinct root causes,
all easy to fix.

This plan enumerates each warning with file/line, root cause,
and the specific fix.

## 2. Goals

- `cargo doc -p nlink --no-deps` emits **zero warnings** after
  this plan ships.
- Add a `cargo doc` step to the rust.yml CI workflow that
  fails on `--deny=rustdoc::broken_intra_doc_links` so this
  class of regression can't recur. (Plan 144 §4.1 reserves the
  `doc` job slot in the new CI shape; this plan owns the
  contents.)

## 3. The 7 warnings — full inventory

Verified locally at commit `9184958` (master tip post-`cargo
fmt`). Each entry: file:line, exact link text, root cause, fix.

### 3.1 — `crates/nlink/src/netlink/impair.rs:26`

```text
warning: unresolved link to `DEFAULT_ASSUMED_LINK_RATE_BPS`
  --> crates/nlink/src/netlink/impair.rs:26:7
   |
26 | //! [`DEFAULT_ASSUMED_LINK_RATE_BPS`], ~80 Gbps), which effectively disables
```

**Root cause**: typo. The actual constant is
`DEFAULT_ASSUMED_LINK_RATE` (no `_BPS` suffix), defined at
`impair.rs:100`:

```rust
pub const DEFAULT_ASSUMED_LINK_RATE: Rate = Rate::bytes_per_sec(10_000_000_000);
```

**Fix**:

```rust
//! [`DEFAULT_ASSUMED_LINK_RATE`], ~80 Gbps), which effectively disables
```

**Side note**: the comment also says "~80 Gbps" but the constant
is `10_000_000_000 bytes/sec` = 80 Gbps. So the description is
correct; only the identifier is stale.

### 3.2 + 3.3 — `crates/nlink/src/netlink/ratelimit.rs:744` and `impair.rs:428`

Both lines are identical:

```rust
/// returns an error by default. Pass [`ReconcileOptions::with_fallback_to_apply(true)`]
```

**Root cause**: rustdoc's intra-doc link parser doesn't accept
function-call syntax in link paths. The `(true)` argument list
is not a valid path component. Rustdoc tries to resolve
`ReconcileOptions::with_fallback_to_apply(true)` as a path,
fails, and emits the warning.

The actual method exists at `tc_recipe.rs:50`:

```rust
pub const fn with_fallback_to_apply(mut self, v: bool) -> Self {
```

**Fix** (both occurrences, same shape):

```rust
/// returns an error by default. Pass [`ReconcileOptions::with_fallback_to_apply`]`(true)`
```

The `(true)` lives outside the link, as inline code, which is
the rustdoc-idiomatic way to show argument values without breaking
the link parser.

### 3.4 - 3.7 — `crates/nlink/src/netlink/messages/tc.rs:166,175,181,189`

All four are the same shape:

```rust
/// Get the handle (qdisc/class ID) as a typed [`TcHandle`].
/// a [`TcHandle`]. Prefer [`handle`](Self::handle) unless you need the
/// Get the parent handle as a typed [`TcHandle`].
/// a [`TcHandle`]. Prefer [`parent`](Self::parent) unless you need the
```

**Root cause**: `TcHandle` is defined in `crate::netlink::tc_handle`
and re-exported at the crate root as `nlink::TcHandle`. From
inside `crate::netlink::messages::tc`, the unqualified
`TcHandle` is not in scope as a path — the import only happens
in `lib.rs` for the public surface.

**Fix** (four occurrences): use the absolute crate path.

```rust
/// Get the handle (qdisc/class ID) as a typed [`crate::TcHandle`].
```

Or, equivalently and more readable in rustdoc UI:

```rust
/// Get the handle (qdisc/class ID) as a typed [`TcHandle`](crate::TcHandle).
```

The second form keeps the displayed text as `TcHandle` (no
`crate::` prefix in the rendered output) while making the path
resolvable. **Pick the second form for all four** — it matches
the surrounding intra-doc-link style in the file (where
`[`handle`](Self::handle)` already uses the `[text](path)`
shape).

## 4. Implementation plan

Single-PR-able. ~6 line edits across 3 files.

### Phase 1: apply the 7 fixes

1. **`crates/nlink/src/netlink/impair.rs:26`** — `DEFAULT_ASSUMED_LINK_RATE_BPS` → `DEFAULT_ASSUMED_LINK_RATE`
2. **`crates/nlink/src/netlink/impair.rs:428`** — `[\`ReconcileOptions::with_fallback_to_apply(true)\`]` → `[\`ReconcileOptions::with_fallback_to_apply\`]\`(true)\``
3. **`crates/nlink/src/netlink/ratelimit.rs:744`** — same as #2
4. **`crates/nlink/src/netlink/messages/tc.rs:166`** — `[\`TcHandle\`]` → `[\`TcHandle\`](crate::TcHandle)`
5. **`crates/nlink/src/netlink/messages/tc.rs:175`** — same
6. **`crates/nlink/src/netlink/messages/tc.rs:181`** — same
7. **`crates/nlink/src/netlink/messages/tc.rs:189`** — same

### Phase 2: add CI gate

8. Add `cargo doc -p nlink --no-deps` step to `.github/workflows/rust.yml`
   (Plan 144 §4.1 reserves the `doc` job slot; this plan provides
   the contents):

   ```yaml
   doc:
     runs-on: ubuntu-latest
     steps:
       - uses: actions/checkout@v4
       - name: Build docs (deny broken intra-doc links)
         env:
           RUSTDOCFLAGS: "-D rustdoc::broken_intra_doc_links -D rustdoc::redundant_explicit_links"
         run: cargo doc -p nlink --no-deps --all-features
   ```

   Notes:
   - `--all-features` matches `[package.metadata.docs.rs] all-features = true`
     in `crates/nlink/Cargo.toml` line 54. So the CI job and
     docs.rs build run identical doc compilations.
   - `RUSTDOCFLAGS` is the standard knob; rustdoc has no
     equivalent of `clippy::deny`.

### Phase 3: verify

```bash
cargo doc -p nlink --no-deps                  # 0 warnings
cargo doc -p nlink --no-deps --all-features   # 0 warnings (matches docs.rs)
RUSTDOCFLAGS="-D rustdoc::broken_intra_doc_links" cargo doc -p nlink --no-deps   # 0 errors
```

If any of these fail, the fix in §3 is incomplete.

## 8. Risks

- **Risk**: A different doc-link form gets rendered differently
  on docs.rs vs locally. **Mitigation**: §6 includes a
  post-publish spot-check. If rendering surprises, follow up in
  0.15.2 or 0.16.
- **Risk**: `RUSTDOCFLAGS` denies break a future contributor's
  unrelated change. **Mitigation**: by enforcing zero warnings,
  contributors learn to fix doc links as they go — same pattern
  as `--deny warnings` for clippy.
- **Risk**: The CI `doc` job adds ~30s to PR build time.
  **Mitigation**: parallel-able with other jobs; negligible
  spend.
