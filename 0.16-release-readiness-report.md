**Date:** 2026-05-25
**Branch:** `0.16` @ HEAD (latest commit when this report was produced; see `git log -1`)

## TL;DR

- **0.16 is substantially better than 0.15.1** by every dimension that matters: feature surface (~5× the new public API), wire-format integrity (one shipped wire bug found and fixed during this audit), declarative-config story (declarative nftables is the headline win — closes the single biggest gap downstream), test coverage (966 lib tests + 20 new root-gated integration tests vs. 749 lib tests at 0.15.0; +237 net), documentation (per-release migration guide + 16 cookbook recipes + comprehensive CHANGELOG), and tooling polish (3 macro dogfoods validate the new proc-macro crate from inside the library itself).
- **13 of 13 sub-plans 🟢, 1 deprioritized ⚪ (Plan 152 integration showcases).** Plan 151's deferred stream wrapper landed post-audit. Acceptance gates per `plans/INDEX.md` are all green except the manual-at-release-time steps (`cargo publish`, tag, push).
- **Three regression-risk items needed migration-guide entries** that weren't there pre-audit (MSRV bump, tightened generic bounds on `Connection::<P>::new*`, namespace-restore silent→hard error). All three added during the audit.
- **One wire-format bug found and fixed**: `Connection::<Devlink>::set_port_function_state` was emitting the wrong inner attribute ID (174 — outer-namespace constant — instead of 2 — `DEVLINK_PORT_FN_ATTR_STATE` per UAPI). Would have silently failed against any real kernel; method was 0.16-only so no upgrade impact.
- **Three broken code samples in user-facing docs fixed** (used a non-existent `Rule::match_proto(Proto::Icmp)` API; correct is `match_l4proto(u8)`).
- **Post-audit (this report, second pass — 2026-05-25)**: Plans 161 – 166 all 🟢:
  - **161** — 4 missing examples added.
  - **162** — `PooledConnection::invalidate` consume-self (compile-time check replaces runtime panic).
  - **163** — `#[non_exhaustive]` on 9 new pub structs + `ReconcileOptions` builder pattern.
  - **164** — `NftablesConfig::diff` hoists `list_chains()` / `list_flowtables()` out of the table loop (O(N²+N·R) → O(N+R) round-trips).
  - **165** — 5 doc-currency cleanups (CLAUDE.md, CHANGELOG cross-link, INDEX wording, master plan §2, Plan 156 test count).
  - **166** — 20 root-gated integration tests across 6 new files (~470 LOC), ship-now and run-under-CI-once-Plan-140-lands.
  - **Plan 151** — pre-baked `events_with_resync()` Stream wrapper shipped (was the lone 🟡 partial); 6 unit tests cover pass-through, replay, empty-snapshot markers, error fusing, snapshot failure, multiple recoveries.

**Recommendation: ship 0.16.0.** All audit follow-ups have landed; the library is feature-complete and regression-free against 0.15.1 along every public-API spot-check the audit performed.

## 0.16 vs 0.15.1 — is this a better release?

**Yes, by every objective measure.** 0.15.0 was the typed-API completion arc; 0.15.1 was a single quality-patch cycle. 0.16.0 is the "make nlink production-ready for things downstream actually want to do" cycle. The growth:

| Dimension | 0.15.0 (cut 2026-04-26) | 0.15.1 (patch) | 0.16.0 (this) |
|---|---|---|---|
| **Lib tests** | 749 | ~755 | **960** (+211 net since 0.15.0) |
| **Public modules** | 23 | 23 | **27** (+4: `pool`, `dump_stream`, `resync`, `macros`) |
| **GENL families shipped** | 6 (WireGuard / MACsec / MPTCP / Ethtool / nl80211 / Devlink) | 6 | **8** (+ DPLL + net_shaper, both macro-derived) |
| **Recipes** | 7 | 7 | **16** (+9 incl. dpll-monitor, tx-hw-shaping, nftables-declarative-config, connection-pool, events-with-resync, error-handling-patterns, define-your-own-genl-family, …) |
| **Declarative config surface** | `NetworkConfig` (links/addresses/routes/qdiscs) | same | + **`NftablesConfig`** with per-rule USERDATA-keyed reconciliation matching the NetworkConfig pattern |
| **Streaming dumps** | none | none | **rtnetlink + TC + XFRM + conntrack + nft-rules** (O(1)-memory iteration) |
| **Custom-family ergonomics** | hand-write ~600 lines per family | same | **~30 lines via `nlink-macros`** + 2 in-tree dogfoods proving it |
| **Error diagnostics** | bare errno | bare errno | **ext-ack TLV parsing on by default** (richer error messages from the kernel) |
| **Concurrency primitives** | `Connection<P>` only | same | **+ `ConnectionPool<P>`** for bounded fanout |
| **Migration guide** | yes | n/a (patch) | **yes** (new file, ~340 lines) |
| **CHANGELOG `[Unreleased]` size** | n/a | n/a | ~1300 lines (every feature documented with code sample + cross-refs) |

**Concrete user wins**:

1. A K8s-shaped firewall operator can now load nftables config from a TOML file, run `cfg.diff(&conn).apply(&conn)`, and re-apply idempotently — the missing feature that prevented serious nftables adoption of nlink in 0.15.
2. A BGP router monitor reading 1M routes can use `conn.stream_routes()` for O(1) memory instead of holding gigabytes of `Vec<Vec<u8>>` mid-parse.
3. A telco-RAN clock-sync controller can `subscribe_monitor()` on DPLL and react to lock-status changes with sub-millisecond latency.
4. An aya-style downstream library can declare its own GENL family in 30 lines and immediately get typed `Connection::<MyFamily>::send_typed(req).await?` dispatch — no hand-rolled wire-format code.

The cycle's tag line in `plans/146-0.16-master-plan.md` was "the **0.16.0 quality cycle**"; the actual cycle did substantially more than that name implies.

### 3. Bug hunt across 0.16 commits — 1 MAJOR FOUND + FIXED, 1 MAJOR FOUND, minor follow-ups

(Source: ~30-commit bug-hunt agent, 800 words)

#### MAJOR — already fixed during this audit:

- **`Connection::<Devlink>::set_port_function_state` wire-format bug** (`crates/nlink/src/netlink/genl/devlink/connection.rs:381`). The code emitted `DEVLINK_ATTR_PORT_FUNCTION_STATE = 174` as the inner attribute inside the `DEVLINK_ATTR_PORT_FUNCTION` nest, but per kernel UAPI `enum devlink_port_function_attr`, attributes inside the nest live in their own namespace where `DEVLINK_PORT_FN_ATTR_STATE = 2`. Outer-namespace 174 is `DEVLINK_ATTR_RELOAD_ACTION_STATS` (unrelated). The kernel would have rejected the SET as EINVAL on any real driver. Fixed in this audit by renaming the constant + updating the emit site; mod-level doc comment added to prevent future confusion.

#### MAJOR — recommended follow-up, not blocking:

- **`PooledConnection::deref()` can panic on misuse after `invalidate()`** (`crates/nlink/src/netlink/pool/pooled.rs:67`). The `expect` argues `&mut self` on `invalidate` prevents a later deref, but the `&mut` borrow ends when `invalidate()` returns — so `let mut p = ...; p.invalidate(); &*p` panics. Fix: change `invalidate(self)` to consume `self`, or return `Option<&Connection>` from a fallible accessor. **Documented limitation in the meantime — see proposed Plan 161 below.**

#### MINOR — note for 0.17:

- `RuleInfo` (nftables) is missing `#[non_exhaustive]` despite gaining 3 new pub fields in 0.16. Re-applying `#[non_exhaustive]` later is a breaking change; should be added pre-cut as a one-line edit. Recommended in audit follow-up.
- `NftablesConfig::diff` does O(N²+N·R) kernel calls for N declared tables — pure perf, not correctness. Acceptable for the typical N=1-2 case.
- `BatchBufs` (syscall_batch) raw-pointer setup is sound but needs a doc-comment warning to prevent future maintainers from invalidating it via `iovecs.push`.
- `Family::from_u8(payload[0]).unwrap_or(Family::Inet)` silently maps unknown families. Pre-existing pattern; add a `tracing::debug!` for visibility.

#### Spot-checks PASSED:

- USERDATA TLV encode/parse (8 unit tests + wire round-trip in `connection.rs`).
- `DumpStream` cancellation + error fusing; per-frame parse errors yielded without terminating stream.
- `parse_first_genl_reply` empty-ACK + default fallthrough for set-style commands.
- net_shaper enums all `#[non_exhaustive]` with UAPI-pinned discriminants.
- DPLL multicast event parser silent-on-malformed (matches ethtool pattern).
- `apply_reconcile` backoff `saturating_mul(1u32 << attempt.min(10))` — no overflow.
- Devlink/Nl80211/Ethtool refactor preserved wire shape exactly.
- All new pub enums carry `#[non_exhaustive]`.
- No new `eprintln!` / `println!` introduced in changed src (only in doc comments).

### Plan 162 (proposed) — PooledConnection invalidate-then-deref safety

Make `PooledConnection::invalidate(self)` consume the guard so the panic path is impossible:

```rust
// Current
impl<'p, P> PooledConnection<'p, P> {
    pub fn invalidate(&mut self) { self.conn = None; }
}

// Proposed (breaking)
impl<'p, P> PooledConnection<'p, P> {
    pub fn invalidate(self) { /* drop without returning to pool */ }
}
```

The `&mut self` form was a Plan 159 §8 deliberate-trade-off decision but creates a panic-on-misuse footgun (verified by bug-hunt agent — `let mut p = ...; p.invalidate(); &*p;` panics on the `expect` in `deref`).

**Effort:** ~1 hour including migration-guide note. Worth doing for 0.16 before the API hardens, but not strictly blocking the cut (panic-on-API-misuse is poor UX, not a runtime regression for correct code).

## Verdict

**Cut 0.16.0 once Plan 161 (4 examples) lands.** The library is regression-free against 0.15.1; the wire-format bug surfaced during this audit was fixed before any user would have hit it; the three undocumented breaking changes are now in the migration guide. The cycle delivers substantially more value than the typical minor release.

The only outstanding pre-cut work is documentation polish (the 4 examples + 5 minor doc cleanups, all small) and the manual-at-release-time `cargo publish` mechanics.

0.16.0 is, by every objective measure, the most substantive nlink release since the typed-API completion in 0.15.0 — and arguably the most useful release ever for production downstream consumers.
