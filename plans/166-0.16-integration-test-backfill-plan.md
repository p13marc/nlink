# Plan 166 — integration-test backfill for 0.16 features

The pre-release audit's plan-verification agent found that 5
0.16 plans shipped without their promised integration tests
(all explicit project-wide deferrals behind the Plan 140 /
Plan 142 privileged-CI gate, but worth tracking as a
single 0.17 backfill effort once that gate lands).

The library code is correct by construction (unit tests +
manual `--apply` runner verification + this audit's spot-checks).
What's missing is the regression net: a downstream PR that
silently breaks the wire format of an integration scenario
wouldn't be caught until manual verification.

## 2. Why this lands in 0.16 (revised — Plan 140 already shipped)

The original framing of this plan ("blocked on Plan 140") was
incorrect: a `cargo public-api` follow-up check (2026-05-25, see
commit `88e3fa2`) plus a CLAUDE.md re-read revealed that
**Plan 140's privileged-CI workflow shipped in 0.15.0** —
`.github/workflows/integration-tests.yml` runs the full
integration-test suite on every push/PR to master under a
container with `CAP_NET_ADMIN` + `CAP_SYS_ADMIN` +
`seccomp=unconfined` + bind-mounted `/lib/modules`. So there is
no privileged-CI prerequisite — the gate has been in tree for
the whole 0.16 cycle, just unused for the new-in-0.16 scenarios
until this plan filled in the test code.

Implications:

- These tests activate the moment 0.16 merges to master — the
  Plan 140 workflow picks them up automatically.
- The maintainer's draft `0.16 → master` PR (PR #3) runs them
  on every push to `0.16` already.
- If any of the 20 new tests are flaky or have a subtle bug,
  the next push to `0.16` will surface it via PR #3's CI run.
  (Timing-dependent tests under §5.1 are the most likely
  candidates — see §9 risk note.)

## 5. Per-test design

### 5.1 Plan 148 — `wait_link_up` + `get_link_stats`

```rust
#[tokio::test]
async fn wait_link_up_returns_when_link_comes_up() {
    nlink::require_root!();
    let lab = LabNamespace::create("wait_link_up_test").await.unwrap();
    let conn = lab.connection::<Route>().await.unwrap();
    let dummy = conn.add_link(LinkBuilder::dummy("d0").build()).await.unwrap();

    // Spawn the wait + bring up in parallel.
    let wait_task = tokio::spawn(async move {
        let conn = lab.connection::<Route>().await.unwrap();
        conn.wait_link_up("d0", Duration::from_secs(5)).await
    });
    tokio::time::sleep(Duration::from_millis(100)).await;
    conn.set_link_up("d0").await.unwrap();
    assert!(wait_task.await.unwrap().is_ok());
}

#[tokio::test]
async fn wait_link_up_times_out() {
    nlink::require_root!();
    let lab = LabNamespace::create("wait_link_up_timeout").await.unwrap();
    let conn = lab.connection::<Route>().await.unwrap();
    conn.add_link(LinkBuilder::dummy("d0").build()).await.unwrap();

    // Don't bring it up — expect timeout.
    let result = conn.wait_link_up("d0", Duration::from_millis(200)).await;
    assert!(result.is_err());
    assert!(result.unwrap_err().is_timeout());
}

#[tokio::test]
async fn get_link_stats_returns_zero_on_unused_dummy() {
    nlink::require_root!();
    let lab = LabNamespace::create("link_stats_test").await.unwrap();
    let conn = lab.connection::<Route>().await.unwrap();
    conn.add_link(LinkBuilder::dummy("d0").build()).await.unwrap();
    let stats = conn.get_link_stats("d0").await.unwrap();
    assert_eq!(stats.tx_packets, 0);
    assert_eq!(stats.rx_packets, 0);
}
```

### 5.2 Plan 149 — streaming dump roundtrip

```rust
#[tokio::test]
async fn stream_links_collects_all_dumped_frames() {
    nlink::require_root!();
    let lab = LabNamespace::create("stream_links_test").await.unwrap();
    let conn = lab.connection::<Route>().await.unwrap();

    for i in 0..5 {
        conn.add_link(
            LinkBuilder::dummy(&format!("d{i}")).build(),
        ).await.unwrap();
    }

    use tokio_stream::StreamExt;
    let mut count = 0;
    let mut stream = conn.stream_links().await.unwrap();
    while let Some(_) = stream.next().await {
        count += 1;
    }
    // Loopback + 5 dummies = at least 6.
    assert!(count >= 6, "expected ≥6 links, got {count}");
}
```

## 9. Risks

- **Plan 140 latency**: Plan 140 has been documented as
  "maintainer-paced." If 140 doesn't land in 0.17's window,
  Plan 166 slips to 0.18.
- **Test flakes from race conditions**: the `wait_link_up`
  test in particular has a "spawn + sleep + bring up" race.
  Mitigation: use `tokio::sync::Notify` or similar to ensure
  the wait task is in the wait state before bringing the link
  up.
- **Hardware-gated test coverage**: XFRM offload, devlink
  rate, net_shaper all need real NICs. CI runners likely
  don't have them. Mitigation: skip-if-absent + document in
  the README that these need a hardware-rich runner pool;
  acceptable since the alternative (no test) is what we
  have today.
- **Lab namespace cleanup**: stale tests can leak netns.
  Mitigation: `LabNamespace::Drop` already handles cleanup;
  add a CI step that asserts no `nlink_test_*` namespaces
  remain after the test run.
