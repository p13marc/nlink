# Plan 155 — neli-parity small features for 0.16

A bag of small additive items where the neli comparison surfaced
real gaps. None requires architectural change; each is a focused
addition with a clear user benefit.

Smaller than Plan 154 (the macros centerpiece), but worth its own
plan because the items share a common theme: **kernel-side
diagnostic surfaces that nlink doesn't currently expose to users.**

## 1. Summary

| # | What | Effort |
|---|---|---|
| 1.1 | Parse extended-ack (NLMSGERR_ATTR_*) TLVs from kernel error responses; surface in `Error::Kernel*` | 4 hours |
| 1.2 | `Connection::<P>::enable_strict_checking(bool)` setsockopt method | 1 hour |
| 1.3 | `Connection::<P>::set_ext_ack(bool)` toggle (currently on by default, no user control) | 30 min |
| 1.4 | Document the existing `*_by_index` ergonomics as a deliberate design choice (the comparison flagged its absence, but the design exists; the doc story doesn't) | 1 hour |

Total: ~6-7 hours.

## 2. Background

The neli comparison (`COMPARISON_NELI.md`) §5 item 5:

> **Extended ACK / strict checking as first-class methods.**
> `rtnl.enable_ext_ack(true)?; rtnl.enable_strict_checking(true)?;`
> — one line each.

Audit results (verified during plan-writing):

- **Ext-ACK is enabled by default** (`socket.rs:192`:
  `socket.set_ext_ack(true).ok();`). But:
  - No user-facing toggle (you can't disable it)
  - **Kernel error responses' extended-ack TLVs are not parsed**
    — the kernel sends `NLMSGERR_ATTR_MSG` (a human-readable error
    string), `NLMSGERR_ATTR_OFFS` (offset into the offending
    message), and `NLMSGERR_ATTR_COOKIE`. None of this surfaces in
    nlink's `Error::Kernel*` variants today; users see
    `errno = 22` with no context, when the kernel was actively
    telling us "attribute X at offset Y was invalid: <reason>"

- **Strict checking** (`NETLINK_GET_STRICT_CHK` sockopt, kernel
  5.0+) is **not** present. Strict checking makes the kernel
  validate dump filters and return an error if they reference
  unknown attributes — useful for catching client/kernel-version
  mismatches early.

The biggest win here is **extended-ack TLV parsing** (item 1.1).
That turns errno-only errors into messages like:
`"Kernel error: Numerical result out of range (errno 34): attribute
TCA_PRIO_MQ_OPT requires u8, got 4 bytes"` — actionable diagnostic
information that exists in the wire response but is currently
thrown away.

## 3. Scope

### In

- Parse `NLMSGERR_ATTR_*` TLVs from kernel error responses.
- Extend `Error::Kernel` / `Error::KernelWithContext` to carry the
  parsed extended-ack message.
- Format extended-ack messages in `Display` for the existing error
  variants so users see them in logs.
- `Connection::enable_strict_checking(bool)` setsockopt method.
- `Connection::set_ext_ack(bool)` toggle (default: on, matching
  current behavior).
- Recipe-level doc for the existing `*_by_index` ergonomics —
  COMPARISON_NELI.md §6 flagged its absence, but the design is
  there; the doc story can be tightened.
- CHANGELOG entries.

### Out

- Strict-checking as on-by-default. It changes behavior on
  kernel-version-mismatch scenarios; opt-in for 0.16, decide
  default for 1.0.
- Re-exposing `NlmF` flag composition (`NLM_F_CREATE | NLM_F_EXCL`
  patterns). COMPARISON_NELI.md §6.7 called the current "flags are
  hidden inside typed methods" pattern a **strength**, not a gap —
  leave it.
- Refactoring `Error` into a deeper variant hierarchy. The current
  `is_X()` predicate pattern is fine; ext-ack just adds a `message:
  Option<String>` field.

## 4. Approach — per item

### 4.1 Parse extended-ack TLVs from error responses

#### Kernel ABI

Per `include/uapi/linux/netlink.h`:

```c
struct nlmsgerr {
    int error;
    struct nlmsghdr msg;
    /* extended ack TLVs follow */
};

enum nlmsgerr_attrs {
    NLMSGERR_ATTR_UNUSED,
    NLMSGERR_ATTR_MSG,         /* NLA_STRING — human-readable error */
    NLMSGERR_ATTR_OFFS,        /* NLA_U32 — offset of bad attr in original msg */
    NLMSGERR_ATTR_COOKIE,      /* NLA_BINARY — opaque cookie for matching */
    NLMSGERR_ATTR_POLICY,      /* NLA_NESTED — policy info */
    NLMSGERR_ATTR_MISS_TYPE,   /* NLA_U32 — missing attr type */
    NLMSGERR_ATTR_MISS_NEST,   /* NLA_U32 — missing nested attr */
};
```

The TLVs sit after the embedded `nlmsghdr` in the error response.
Their presence is signaled by the `NETLINK_CAP_ACK` and
`NETLINK_EXT_ACK` sockopts being set on the listening socket (both
already enabled by default in nlink's `socket.rs:192`).

#### Code change

Extend `crates/nlink/src/netlink/error.rs`:

```rust
#[derive(Debug)]
#[non_exhaustive]
pub enum Error {
    // ... existing ...
    Kernel {
        errno: i32,
        message: String,
        /// Extended-ack human-readable error message from the
        /// kernel, when available. Kernel 4.12+ provides this for
        /// most subsystems with NETLINK_EXT_ACK enabled.
        ext_ack: Option<String>,
        /// Offset (in bytes) into the original request where the
        /// kernel detected the problem. Useful for debugging
        /// attribute-encoding bugs.
        ext_ack_offset: Option<u32>,
    },
    KernelWithContext {
        operation: String,
        errno: i32,
        message: String,
        ext_ack: Option<String>,
        ext_ack_offset: Option<u32>,
    },
}

impl std::fmt::Display for Error {
    fn fmt(&self, f: &mut std::fmt::Formatter) -> std::fmt::Result {
        match self {
            Error::KernelWithContext { operation, errno, message, ext_ack, .. } => {
                write!(f, "{operation}: {message} (errno {errno})")?;
                if let Some(msg) = ext_ack {
                    write!(f, ": {msg}")?;
                }
                Ok(())
            }
            // ... etc
        }
    }
}
```

Extend `crates/nlink/src/netlink/message.rs` (or wherever
`NlMsgError::from_bytes` lives) to parse the trailing TLVs after the
embedded `nlmsghdr`. Each TLV is a standard nlattr; reuse `AttrIter`.

```rust
pub struct NlMsgError {
    pub error: i32,
    pub msg_hdr: NlMsgHdr,
    pub ext_ack_msg: Option<String>,
    pub ext_ack_offset: Option<u32>,
    // pub ext_ack_cookie: Option<Vec<u8>>,  // omit for now; niche
    // pub ext_ack_miss_type: Option<u32>,   // omit for now; niche
    // pub ext_ack_miss_nest: Option<u32>,   // omit for now
}

impl NlMsgError {
    pub fn from_bytes(payload: &[u8]) -> Result<Self> {
        // Existing: parse error code + nlmsghdr.
        // New: iterate any trailing TLVs.
        let mut ext_ack_msg = None;
        let mut ext_ack_offset = None;
        let tlv_offset = std::mem::size_of::<i32>() + std::mem::size_of::<NlMsgHdr>() /* + alignment */;
        if payload.len() > tlv_offset {
            for (attr_type, attr_payload) in AttrIter::new(&payload[tlv_offset..]) {
                match attr_type {
                    1 /* NLMSGERR_ATTR_MSG */ => {
                        ext_ack_msg = Some(
                            std::str::from_utf8(attr_payload)
                                .unwrap_or("")
                                .trim_end_matches('\0')
                                .to_string(),
                        );
                    }
                    2 /* NLMSGERR_ATTR_OFFS */ if attr_payload.len() >= 4 => {
                        ext_ack_offset = Some(u32::from_ne_bytes(
                            attr_payload[..4].try_into().unwrap(),
                        ));
                    }
                    _ => {}
                }
            }
        }
        // ... return Self ...
    }
}
```

Propagate `ext_ack_msg` + `ext_ack_offset` through `Error::from_errno`
(or whichever construction path is used in `send_ack`).

#### Tests

In `crates/nlink/src/netlink/error.rs::tests`:

- Synthesize a kernel error response with extended-ack TLVs;
  verify the parsed `Error` carries the message and offset.
- Round-trip Display: format includes the ext-ack message when

In `tests/integration/`: any operation that's known to trigger a
descriptive kernel error (e.g. `add_link` with a conflicting name
when the kernel provides a useful message). Verify the rendered

### 4.2 `enable_strict_checking(bool)` setsockopt

```rust
// crates/nlink/src/netlink/connection.rs

impl<P: ProtocolState> Connection<P> {
    /// Enable kernel-side strict checking (`NETLINK_GET_STRICT_CHK`,
    /// kernel 5.0+). When enabled, the kernel validates dump request
    /// filters strictly and returns an error if they reference
    /// unknown attributes. Useful for catching client/kernel-version
    /// mismatches early.
    ///
    /// Off by default for backwards compatibility with older kernels
    /// (the sockopt returns ENOPROTOOPT on pre-5.0 kernels; this
    /// method returns Ok(()) silently in that case to remain
    /// portable).
    pub fn enable_strict_checking(&self, on: bool) -> Result<()> {
        self.socket.set_strict_checking(on)
    }
}
```

Implement `NetlinkSocket::set_strict_checking` to call setsockopt
with `NETLINK_GET_STRICT_CHK` (= 12). Return Ok on ENOPROTOOPT
(graceful degradation on old kernels).

```rust
impl<P: ProtocolState> Connection<P> {

    /// 4.12+). Enabled by default — disabling is rarely needed,
    /// exposed for parity with neli's API and for users who want

        self.socket.set_ext_ack(on)

}
```

### 4.4 Document `*_by_index` design intent

COMPARISON_NELI.md §6 said:

> No ifindex-first / namespace-aware ergonomics. Examples like
> `getlink.rs` walk `Rtm::Getlink` dumps by hand and string-match
> `Ifla::Ifname`. There is no `get_link_by_index` typed helper.

That's true for neli. nlink **has** the `*_by_index` ergonomics
already (Plan 148 §4.4 was about doc'ing them better). The
documentation hasn't yet caught up to call this out as a
deliberate design choice that distinguishes nlink from neli.

Update CLAUDE.md's "Connection patterns" section to add:

```markdown
## Namespace-safe APIs

nlink provides `*_by_index` variants alongside `*_by_name` for every
resource lookup. The `_by_index` variants take a kernel ifindex
directly, so they're safe to call from any process mount-namespace
(the index is always relative to the connection's netns). The
`_by_name` variants read `/sys/class/net/` from the calling
process's mount namespace — convenient for simple cases, but
surprises inside foreign netns. **For namespace-aware code (CNI
plugins, multi-tenant managers), prefer the `_by_index` variants
or pre-resolve names once via `get_link_by_name` then pass the
index to subsequent calls.**

This is a deliberate design choice. `neli` and `vishvananda/netlink`
both leave namespace handling to the caller, which is a documented
footgun in [Cilium issue #40280](https://github.com/cilium/cilium/issues/40280).
nlink's typed `InterfaceRef::Index(u32)` and the per-method
`_by_index` variants make namespace-correct code natural to write.
```

Also add a "Namespace safety" section to the top of the rustdoc
landing page (`crates/nlink/src/lib.rs` `//!` block).

Plan 148 §4.4 already covers per-method doc blocks; this 4.4 item
covers the **strategic positioning** at the docs level.

#### Effort

1 hour.

## 8. Risks

- **Error variant field additions**: technically a public-API
  change. Mitigated by `#[non_exhaustive]` on the enum (which
  nlink already uses per `error.rs` audit). Adding fields to a
  non_exhaustive variant is officially semver-additive per the
  Cargo book.
- **Kernel-version variance**: `NETLINK_GET_STRICT_CHK` is 5.0+;
  `NETLINK_EXT_ACK` is 4.12+. Both methods return Ok silently
  when the sockopt isn't supported (ENOPROTOOPT). Document
  per-method.
- **Ext-ack offset interpretation**: the offset is into the
  original request the user sent, not into the response. For the
  user to find the byte at that offset, they'd need to know what
  they sent — currently not preserved. The offset is informational;
  document that.
- **Integration-test brittleness**: kernel ext-ack message text is
  not API-stable across versions. Test should assert the message
  is non-empty and contains a substring like "rejected" or
  "invalid", not match exact text.
