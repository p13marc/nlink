# Plan 203 — Master: second-audit findings batch

## 1. Context

The 0.19 cycle's first audit shipped `5ef0808` (5 fixes + 18 tests).
A second-pass audit (`docs/AUDIT_REPORT_2026_05_31.md`) found
~96 distinct bugs across angles the first audit didn't reach.
This master orchestrates the 13 sub-plans that close every
**CRITICAL** and **HIGH** finding plus the high-value MEDIUM items
before 0.19 cuts.

The user has explicitly authorized breaking changes for this cycle.
Several plans take that authorization to remove dead-code knobs
(C5), tighten type-level wire correctness (M11), and gate buggy
silent-default patterns in bins (H5/H6) — all of which would
otherwise have been deferred for semver reasons.

## 2. Why this fits in 0.19 (not 0.20)

The five CRITICAL findings are wire-format defects that ship today
silently broken on real kernels (NFT verdicts, XFRM policy
add/del, devlink event subscription, NetworkConfig purge). Each one
is a one-line / one-field change. Shipping these as 0.19 means:

- No user has stable code depending on the broken behavior.
- The 0.19 → 0.19.1 conversation becomes "we found and fixed five
  silent-corruption bugs in cycle review" — much better than
  "0.19 shipped with five known broken APIs."
- 0.20 stays open for the larger architectural plans (Plan 197
  ovpn, Plan 198 nftables sets, Plan 195 reflectors).
