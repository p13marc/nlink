# Plan 153 — kernel feature bundle for 0.16

Three independent kernel features, each ~1-2 days, bundled into one
plan because they share characteristics (small surface, real demand
from specific user segments, no architectural overlap) and benefit
from a single kernel-version probing + CI modprobe wave update.

Bundled rather than separate plans because each is too small to merit
its own master/sub-plan structure and they'd each repeat the same
"add a typed builder, wire a Connection method, write an integration
test, update modprobe wave" pattern.

## 1. Summary

| # | Feature | Kernel | Audience | Effort |
|---|---|---|---|---|
| 1.1 | XFRM IPsec offload (`XFRMA_OFFLOAD_DEV`) | 6.11+ (IPv6 ESP + IPv4 UDP-encap ESP); offload itself older | IPsec at scale (cloud, telco) | 1 day |
| 1.2 | Devlink rate + port-function-state | 5.x baseline | Cloud + SmartNIC users | 2 days |
| 1.3 | TX H/W shaping via `net_shaper` GENL | 6.13+ | Time-sensitive networking, traffic shaping operators | 2 days |

Total: ~5 days.

## 2. Goals

By end of plan:

- **XFRM offload**: users can request hardware offload on an XFRM SA
  via `.offload(dev, flags)` on the existing `XfrmSaBuilder`. Lib
  emits `XFRMA_OFFLOAD_DEV` correctly. Kernel acceptance verified
  on at least one offload-capable NIC (mlx5 in CI is a stretch
  goal; verify by `--apply` example on dev hardware).
- **Devlink rate + port-function-state**: typed `DevlinkRateBuilder`
  + `set_port_function_state(...)` methods. Cloud users can rate-limit
  SR-IOV VFs and toggle port functions through nlink.
- **TX H/W shaping**: typed `NetShaperBuilder` + corresponding
  `Connection<NetShaper>` methods for the new (kernel 6.13) generic
  netlink interface.

## 3. Scope

## 4. Per-feature design

### 4.1 XFRM IPsec offload

#### Kernel ABI

```c
// include/uapi/linux/xfrm.h
enum xfrm_attr_type_t {
    /* ... */
    XFRMA_OFFLOAD_DEV,       // struct xfrm_user_offload
};

struct xfrm_user_offload {
    int ifindex;
    __u8 flags;
};

#define XFRM_OFFLOAD_IPV6       0x1
#define XFRM_OFFLOAD_INBOUND    0x2
#define XFRM_OFFLOAD_PACKET     0x4   // kernel 6.0+
```

The attribute is a 5-byte struct (4 bytes ifindex + 1 byte flags),
appended to the `RTM_NEWSA` (add) message. Replace, update, and
get-by-id are unchanged.

#### nlink shape

Extend `crates/nlink/src/netlink/xfrm.rs`:

```rust
impl XfrmSaBuilder {
    /// Request hardware offload on this SA. `dev` is the device to
    /// offload to (resolved to ifindex). `flags` is a bitmap of
    /// `XfrmOffloadFlag` values.
    ///
    /// Kernel 6.11+ extended offload to IPv6 ESP and IPv4 UDP-encap
    /// ESP; earlier kernels support IPv4 ESP only. NIC support varies
    /// — mlx5, hns3, and a few others; check `ethtool -k <dev> | grep
    /// esp`.
    ///
    /// Failure mode: if the NIC doesn't support the requested offload
    /// shape, the kernel returns EOPNOTSUPP on the add. The lib
    /// surfaces that as a normal Error::Kernel; callers must decide
    /// whether to retry without offload.
    pub fn offload(
        mut self,
        dev: impl Into<InterfaceRef>,
        flags: XfrmOffloadFlag,
    ) -> Self {
        self.offload = Some((dev.into(), flags));
        self
    }
}

bitflags! {
    pub struct XfrmOffloadFlag: u8 {
        const IPV6     = 0x1;
        const INBOUND  = 0x2;
        const PACKET   = 0x4;  // kernel 6.0+
    }
}
```

The Connection's `add_sa` flow resolves `dev` to ifindex (reuse the
existing namespace-aware lookup) and emits `XFRMA_OFFLOAD_DEV`.

#### Tests

`tests/integration/xfrm.rs` (if exists; else create):

```rust
#[tokio::test]
async fn test_xfrm_sa_with_offload_flag_kernel_accepts() {
    require_root!();
    nlink::require_module!("xfrm_user");
    // Offload requires a real device; use lo as a no-op target.
    // Kernels without offload support return EOPNOTSUPP — accept
    // that as success for this test (we're testing the wire format,
    // not the NIC capability).
    let result = conn.add_sa(/* ... */ .offload("lo", XfrmOffloadFlag::empty())).await;
    match result {
        Ok(()) => {} // accepted
        Err(e) if e.is_not_supported() => {} // expected on most CI kernels
        Err(other) => panic!("unexpected error: {other}"),
    }
}
```

#### Effort

1 day including the integration test and a CHANGELOG note.

### 4.2 Devlink rate + port-function-state

#### Kernel ABI

Devlink GENL family. Commands:

- `DEVLINK_CMD_RATE_NEW`, `DEVLINK_CMD_RATE_DEL`,
  `DEVLINK_CMD_RATE_SET`, `DEVLINK_CMD_RATE_GET`
- `DEVLINK_CMD_PORT_FUNCTION_SET`

Attribute IDs (from `include/uapi/linux/devlink.h`):

```c
DEVLINK_ATTR_RATE_TYPE              = 165, /* u16 */
DEVLINK_ATTR_RATE_TX_SHARE          = 166, /* u64, bytes/sec */
DEVLINK_ATTR_RATE_TX_MAX            = 167, /* u64, bytes/sec */
DEVLINK_ATTR_RATE_NODE_NAME         = 168, /* string */
DEVLINK_ATTR_RATE_PARENT_NODE_NAME  = 169, /* string */
DEVLINK_ATTR_PORT_FUNCTION          = 145, /* nested */
DEVLINK_ATTR_PORT_FUNCTION_STATE    = 174, /* u8 */
```

#### nlink shape

```rust
// crates/nlink/src/netlink/genl/devlink/types.rs

pub struct DevlinkRate {
    pub bus_name: String,
    pub device_name: String,
    pub node_name: String,
    pub parent_node: Option<String>,
    pub tx_share: Option<Rate>,
    pub tx_max: Option<Rate>,
    pub rate_type: DevlinkRateType,  // Leaf | Node
}

impl DevlinkRate {
    pub fn new(bus: impl Into<String>, dev: impl Into<String>, node: impl Into<String>) -> Self;
    pub fn tx_share(mut self, rate: Rate) -> Self;
    pub fn tx_max(mut self, rate: Rate) -> Self;
    pub fn parent_node(mut self, name: impl Into<String>) -> Self;
}

impl Connection<Devlink> {
    pub async fn add_rate(&self, rate: &DevlinkRate) -> Result<()>;
    pub async fn del_rate(&self, bus: &str, dev: &str, node: &str) -> Result<()>;
    pub async fn set_rate(&self, rate: &DevlinkRate) -> Result<()>;
    pub async fn get_rates(&self, bus: Option<&str>, dev: Option<&str>) -> Result<Vec<DevlinkRate>>;

    pub async fn set_port_function_state(
        &self,
        bus: &str,
        dev: &str,
        port_index: u32,
        state: DevlinkPortFunctionState,
    ) -> Result<()>;
}

pub enum DevlinkPortFunctionState {
    Inactive,
    Active,
}
```

Reuses the existing `nlink::Rate` typed unit (a good fit since
devlink rate fields are bytes/sec).

#### Tests

`tests/integration/genl_devlink.rs`:

- Skip cleanly on hosts without devlink-capable hardware (no
  `/sys/class/devlink/` entries).
- Use `mlxsw_spectrum` or similar simulator if available; else
  document as `#[ignore]` manual-only.

#### Effort

2 days. The devlink module exists in nlink already (lookup
`crates/nlink/src/netlink/genl/devlink/`), so the scaffolding is
in place — this is adding new commands and types, not building from
scratch.

### 4.3 TX H/W shaping via `net_shaper` GENL

#### Kernel ABI

Kernel 6.13 added the `net_shaper` GENL family. Commands and
attributes are documented in
[`Documentation/netlink/specs/net_shaper.yaml`](https://docs.kernel.org/userspace-api/netlink/specs.html).

Conceptual model:
- A "shaper" is a queue-disciplines-adjacent abstraction for NIC
  hardware. Each shaper has a bandwidth, burst, priority, and a
  scope (interface | per-queue | per-group).
- Operations: NEW, DEL, SET, GET, GET-DUMP, CAPS (introspection).

#### nlink shape

This is a new GENL family for nlink. Add a marker type:

```rust
// crates/nlink/src/netlink/protocol.rs

#[derive(Debug, Default, Clone, Copy)]
pub struct NetShaper;

impl ProtocolState for NetShaper { /* GENL family, async constructor */ }
```

New module `crates/nlink/src/netlink/genl/net_shaper/{mod,connection,types}.rs`
mirroring the existing devlink shape:

```rust
pub struct Shaper {
    pub ifindex: u32,
    pub scope: ShaperScope,    // interface | per-queue
    pub bw: u64,               // bytes/sec
    pub burst: Option<u64>,    // bytes
    pub priority: u32,
}

impl Connection<NetShaper> {
    pub async fn add_shaper(&self, shaper: &Shaper) -> Result<()>;
    pub async fn del_shaper(&self, ifindex: u32, queue: Option<u32>) -> Result<()>;
    pub async fn set_shaper(&self, shaper: &Shaper) -> Result<()>;
    pub async fn get_shapers(&self, ifindex: Option<u32>) -> Result<Vec<Shaper>>;
    pub async fn get_caps(&self, ifindex: u32) -> Result<ShaperCaps>;
}
```

#### Tests

`tests/integration/genl_net_shaper.rs`:

- Skip if `net_shaper` GENL family doesn't exist (`genl::resolve_family`
  returns NotFound).
- Skip if no shaper-capable NIC (caps query returns empty).
- Smoke test against any NIC that supports it; mark `#[ignore]` if no
  capable device.

#### Effort

2 days. New GENL family is more setup than devlink (which exists),
but the pattern is established.

## 5. Implementation phases

Each feature = independent PR.

### Phase 1: XFRM offload (smallest; ships first)

1. Extend `XfrmSaBuilder` with `.offload(dev, flags)`.
2. Wire `XFRMA_OFFLOAD_DEV` emission in `add_sa`.
3. Integration test (lenient: accept EOPNOTSUPP as success).
4. CHANGELOG entry.

### Phase 2: Devlink rate + port function state

5. Add `DevlinkRate` struct + builder per §4.2.
6. Add `add_rate` / `del_rate` / `set_rate` / `get_rates` Connection
   methods.
7. Add `set_port_function_state`.
8. Integration test (skip if no devlink hardware).
9. Workflow modprobe: nothing new (devlink is built-in usually).
10. CHANGELOG entry + recipe.

### Phase 3: TX H/W shaping (largest; ships last)

11. Add `NetShaper` protocol marker.
12. Add `crates/nlink/src/netlink/genl/net_shaper/` module.
13. Add `Connection<NetShaper>` methods per §4.3.
14. Integration test (skip on hosts without the family).
15. Workflow modprobe: nothing new (net_shaper is built-in if
    CONFIG_NET_SHAPER=y).
16. CHANGELOG entry + recipe.

## 8. Risks

- **Hardware-test scarcity**: XFRM offload, devlink rate, and shaper
  features all require specific NIC capabilities. CI doesn't have
  any of these. Mitigation: every integration test is permissive on
  EOPNOTSUPP / ENODEV / family-missing; the wire-format correctness
  is what we test in CI. Real hardware validation is manual on the
  maintainer's hardware before the release cut.
- **Devlink ABI churn**: devlink attribute IDs have moved between
  kernel versions. Pin against kernel 6.10+ (which matches our
  `rust:bookworm` CI baseline). Older kernels may reject some
  attributes — document the minimum-kernel-version per feature.
- **`net_shaper` is brand new (6.13)**: ABI might still shift in
  6.14/6.15. Worth checking the YNL spec (`Documentation/netlink/specs/net_shaper.yaml`)
  for any "deprecated" annotations before shipping. If the ABI
  isn't stable yet, ship behind `--cfg nlink_unstable` instead of
  the public surface (gate documented in STRATEGIC_ANALYSIS §4.5).
  Decision: check at implementation time; default to public if
  spec is clean.
- **Bundling-PR review burden**: 5 days of work in 3 features. If
  reviewer bandwidth is limited, the three phases ship as three PRs
  (per §5) rather than one giant PR. Each is independently
  reviewable in a single sitting.

## 9. No out-of-scope follow-ups

Each of these features is a complete kernel-surface addition. The
follow-ups (devlink resource / param / health reporter; xfrm

End of plan.
