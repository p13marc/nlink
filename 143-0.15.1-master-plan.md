# Plan 143 — 0.15.1 patch release master

A regression slipped through 0.15.0: an example that uses
`nlink::lab::with_namespace` doesn't declare
`required-features = ["lab"]` in `crates/nlink/Cargo.toml`. As a
result, **`cargo test --workspace` (default features) fails to
compile** for any downstream consumer or contributor — and the CI
on master is red.

This plan orchestrates the 0.15.1 patch cut: scope, acceptance
gates, sub-plan references, and the release checklist.

## 1. Why a patch release

- **CI is red on master.** Anyone running `cargo test` at the
  workspace root after `git clone` hits a build error. That's
  the worst kind of footgun for a 0-day-after-publish library.
- **The fix is one line of TOML.** A patch cut is the
  proportional response — no API changes, no semver risk.
- **A regression escaped.** The CI workflow we have today
  doesn't catch this class of bug because it runs
  `cargo test --verbose` which fails… but no PR has been opened
  on top of the broken master to surface the failure on a green
  baseline. Add a regression-prevention CI safety net in the same
  release (Plan 144), so the next mistake of this shape can't
  ship.
- **Two doc warnings on `cargo doc` are visible on docs.rs**
  for everyone who lands on the 0.15.0 docs page right now.
  Cheap to fix; ship in the patch (Plan 145).

## 4. Phase 0 — the actual fix (lives in this plan, ship first)

Because the ad-hoc PR is one line, it lives in this master plan
rather than its own sub-plan. Sub-plans 144 and 145 ship after
Phase 0 lands.

### 4.1 Diagnosis

`crates/nlink/examples/xfrm/ipsec_monitor.rs` was promoted from
dump-only to a full lifecycle runner during the 0.15.0 post-cut
tail (commit `5634f1a`). The `--apply` codepath uses
`nlink::lab::with_namespace`. But the corresponding
`[[example]]` entry in `crates/nlink/Cargo.toml` (lines 211-214,
verified) does not declare `required-features = ["lab"]`:

```toml
# XFRM examples (IPsec)
[[example]]
name = "xfrm_ipsec_monitor"
path = "examples/xfrm/ipsec_monitor.rs"
```

Compare to the correct gating on `lab_three_namespace` (lines
295-299):

```toml
[[example]]
name = "lab_three_namespace"
path = "examples/lab/three_namespace.rs"
required-features = ["lab"]
```

The `lab` feature gates `pub mod lab` at `crates/nlink/src/lib.rs:140`
behind `#[cfg(feature = "lab")]`. Without the feature, the
import `use nlink::lab::with_namespace` at
`crates/nlink/examples/xfrm/ipsec_monitor.rs:165` fails to
resolve, and the example fails to compile in default-features
builds.

### 4.2 Fix

Single edit to `crates/nlink/Cargo.toml`:

```toml
# XFRM examples (IPsec)
[[example]]
name = "xfrm_ipsec_monitor"
path = "examples/xfrm/ipsec_monitor.rs"
required-features = ["lab"]   # ADDED
```

That's it. No source code changes.

### 4.3 Verify

```bash
cargo build --workspace                       # was failing; should now pass
cargo test --workspace                        # was failing; should now pass
cargo build -p nlink --example xfrm_ipsec_monitor --features lab   # always worked
cargo build -p nlink --example xfrm_ipsec_monitor                   # NEW: should be a no-op build (example skipped due to required-features)
```

The "should be a no-op build" outcome is correct: cargo silently
skips examples whose required-features aren't enabled. That's the
intended escape hatch for feature-gated examples.

## 5. Acceptance gates for 0.15.1

A 0.15.1 cut requires ALL of:

- [ ] **Phase 0 shipped** — `cargo test --workspace` is green on
      a fresh `git clone`, no features enabled.
- [ ] **Plan 144 shipped** — CI catches example-gating regressions
      automatically; `cargo-semver-checks` and `cargo public-api`
      are wired (and would have caught the original 0.15.0
      `add_class_config` rename bug we surgically fixed pre-publish).
- [ ] **Plan 145 shipped** — `cargo doc -p nlink --no-deps`
      emits zero warnings.
- [ ] **`cargo public-api` diff** between `0.15.0` and the
      `0.15.1` candidate is **empty** (no public API changes in a
      patch release; if the tool reports any, investigate).
- [ ] **CHANGELOG `## [0.15.1]`** lists every change with file
      refs and commit hashes.
- [ ] **Workspace version bumped** to `0.15.1` in `Cargo.toml`.
- [ ] **`cargo publish -p nlink --dry-run`** succeeds without
      warnings about feature, dependency, or metadata issues.
- [ ] **No `docs/migration_guide/0.15.0-to-0.15.1.md`** — patch
      releases don't need migration guides per the convention in
      `docs/migration_guide/README.md` (only minor + major).

## 7. Risks

- **Risk**: A second test-failure-class regression hides somewhere
  no one's run. **Mitigation**: Plan 144 adds
  `cargo test --workspace` with default features as a CI gate, so
  any future `lab`-using example without the feature declaration
  fails fast.
- **Risk**: The `cargo-semver-checks` baseline for `0.15.0`
  reveals an accidental API break that 0.15.0 already shipped.
  **Mitigation**: If it fires, accept and document; the tool is
  for forward-looking enforcement, not retroactive enforcement.
- **Risk**: `cargo public-api` reveals an unintended public item
  added in 0.15.0 (e.g. something that should have been `pub(crate)`).
  **Mitigation**: Same as above — accept the 0.15.0 baseline as
  ground truth, enforce going forward.
